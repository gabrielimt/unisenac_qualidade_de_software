# 1. Diagnóstico da Situação Atual

## 1.1 Papéis atuais identificados

Liste quais papéis vocês acreditam que existem atualmente na startup.

- Desenvolvedor
- Analista de sistemas
- Gerente de produto

---

## 1.2 Quem é responsável pela qualidade hoje?

Descreva como vocês acreditam que a qualidade está sendo tratada atualmente.

> Resposta: Atualmente, não tá claro quem é o responsável por garantir a qualidade do software dentro da startup Local Eats. Isso indica que a validação provavelmente está sendo feita de forma superficial pelos próprios desenvolvedores ou deixada para os clientes descobrirem em produção.

---

## 1.3 Problemas identificados

Liste os principais problemas relacionados à falta de organização da qualidade.

- Clientes relatam erros ao finalizar pedidos
- Restaurantes recebem pedidos duplicados
- Funcionalidades chegam à produção com defeitos

---

## 1.4 Impactos desses problemas

Explique quais são as consequências desses problemas para o sistema e para os usuários.

> Resposta: Os erros na finalização dos pedidos e a duplicidade frustram os clientes e prejuízos operacionais/financeiros para os restaurantes locais. Além disso, a chegada de funcionalidades com defeitos em produção diminui a confiabilidade da plataforma e gera retrabalho constante para equipe técnica.

---

## 1.5 A qualidade é responsabilidade de quem?

Explique se a qualidade deve ser responsabilidade de uma pessoa ou de toda a equipe.

> Resposta: A qualidade de software deve ser entendida como uma responsabilidade compartilhada pela equipe, mesmo quando existe um papel específico de QA.

# 3. Práticas de QA Sugeridas

Sugira práticas que a startup pode adotar para melhorar a qualidade.

---

## 3.1 Lista de práticas

- Testes manuais das funcionalidades principais
- Registro e acompanhamento de bugs
- Testes exploratórios
- Revisão de funcionalidades antes da entrega

---

## 3.2 Explicação das práticas

Explique brevemente cada prática sugerida.

### Prática 1:
> Descrição: Consiste em um humano testar os fluxos mais críticos do sistema, simulando o uso real do cliente (ex: colocar item no carrinho e tentar finalizar a compra).

### Prática 2:
> Descrição: Utilizar uma ferramenta para documentar todo defeito encontrado, descrevendo como reproduzi-lo para que o desenvolvedor possa corrigi-lo.

### Prática 3:
> Descrição: Estabelecer uma etapa obrigatória onde outra pessoa (preferencialmente o QA) verifica o trabalho do desenvolvedor antes que o código vá para o ambiente de produção.

# 4. Anúncios de Contratação

A startup decidiu contratar novos profissionais. Crie anúncios de vagas.

> Mínimo: 2 vagas

---

## 4.1 Vaga 1 – Analista de Qualidade de Software (QA)

### Descrição da vaga
> A Local Eats está buscando um profissional de Qualidade de Software (QA) para apoiar a equipe de desenvolvimento na melhoria contínua da qualidade do nosso sistema de pedidos online. A pessoa atuará colaborando para identificar riscos e planejar testes.

### Responsabilidades
- Planejar e executar testes de software 
- Identificar e registrar defeitos 
- Realizar testes exploratórios nas funcionalidades

### Requisitos obrigatórios
- Conhecimento básico de testes de software 
- Capacidade de identificar defeitos e documentá-los 
- Boa comunicação e trabalho em equipe

### Requisitos desejáveis
- Experiência com testes de aplicações web 
- Conhecimento de ferramentas de gestão de defeitos 
- Noções de automação de testes

### Certificações desejáveis
- ISTQB - CTFL (Certified Tester Foundation Level)

---

## 4.2 Vaga 2 – Desenvolvedor (Foco Backend)

### Descrição da vaga
> Buscamos um Desenvolvedor apaixonado por tecnologia para integrar nossa equipe da Local Eats. O profissional será responsável por estruturar nossa API de pedidos, garantindo performance e confiabilidade na comunicação entre clientes e restaurantes.

### Responsabilidades
- Desenvolver funcionalidades seguras e escaláveis para a plataforma.
- Colaborar com a pessoa de QA para corrigir defeitos de integração e de lógica.
- Melhorar a estrutura de código para evitar problemas de duplicidade de pedidos.

### Requisitos obrigatórios
- Conhecimento básico de desenvolvimento de sistemas 
- Capacidade de trabalhar em equipe para garantir a qualidade conjunta.
- Domínio de linguagens backend (ex: Node.js, Java, C#).

### Requisitos desejáveis
- Conhecimento de versionamento com Git 
- Experiência prévia em plataformas de e-commerce ou delivery.
- Noções de testes unitários.

### Certificações desejáveis
- Nenhuma certificação obrigatória, mas cursos em engenharia de software são um diferencial.
